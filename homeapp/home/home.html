This section is mostly for the geeky among us. If you don't care about how web pages are built, this is definitely a skippable section.

The latest renovation to this home has produced a place which is both a single- and multi-page app.

This has the neat effect of putting all the main elements on one page, shown one after the other, but showing each separately when selected from the menu. Thus, selecting the resume brings up a page with just the navbar, footer and resume.

These segments are built primarily with a mix of AngularJS and w3.CSS frameworks. The last version was built primarily using the Bootstrap and jQuery frameworks. The version before that was almost entirely PHP driven, with custom JavaScripts.

I always want to do things in the most optimal way, so part of my personal challenge is balancing when to pull the trigger on a new version. It is a balancing act of time on hand, time invested and return for the effort.

Once upon a time, I carefully hand-crafted every line of code. No dependencies. Everything from scratch. Often inspired by others, but always worked out and implemented by me, myself and I.

I can be silly sometimes.

There are trade-offs to using the latest fancy buzz-word flavors. CSS frameworks like Bootstrap and w3.CSS have major advantages in the speed of developing pages, for example, but clutter up the html with strings of classes not normally encountered in custom CSS. The purist in me objects, while my practical side says something like, "No problem!".

Unfortunately, unless I point these mechanics out in   code snippets,  almost no one is going to be able to appreciate them but the afore-mentioned me, myself and I. I don't need anyone else's approval, explicit or otherwise, but life is about balance. I am just looking to stay a happy, healthy, well-balanced developer.
